# Normiculus Realtime Database Schema

This document defines the exact schema and business logic for the JSON records stored in the Firebase Realtime Database for the Normiculus Reminder App. 

An executor agent reading the `https://normiculus-default-rtdb.asia-southeast1.firebasedatabase.app/reminders.json` endpoint should use this reference to properly parse, evaluate, and dispatch reminders.

## 1. Top-Level Structure

The database acts as a root object containing two separate nodes: a cache-busting timestamp integer, and a dictionary of `ReminderPayload` objects where the key is a unique ID generated by Firebase (or the frontend).

```json
// GET https://<your-project>.firebasedatabase.app/last_updated.json
// GET https://<your-project>.firebasedatabase.app/reminders.json

{
  "last_updated": 1740815412431, // Integer epoch triggered on ANY change to the tree
  "reminders": {
    "reminderId_1": { /* Reminder Payload */ },
    "reminderId_2": { /* Reminder Payload */ }
  }
}
```

## 2. Reminder Payload Schema

Each Reminder payload conforms to the following strict shape:

```typescript
type ReminderPayload = {
  message: string;
  date: DateConfig;
  times: TimeConfig[];
  time?: TimeConfig;   // DEPRECATED: Retained for backward compatibility with old records.
  contacts?: string[]; // NOTE: Firebase omits this key entirely if the array is empty.
  createdAt: string;   // ISO 8601 Timestamp
}
```

### 2.1 The `message` Field
- **Type**: `string`
- **Constraint**: Maximum length of 1500 characters.
- **Content**: The literal text that should be sent in the reminder payload.

### 2.2 The `date` Field (DateConfig)
Defines the recurrence/frequency rules for the reminder.
The `date.type` enum dictates how the reminder behaves.

```typescript
type DateConfig =  
  | { type: "one_time" }
  | { type: "everyday" }
  | { type: "weekdays" } // Monday through Friday
  | { type: "weekends" } // Saturday and Sunday
  | { type: "custom", customDays: string[] } 
  | { type: "specific_dates", specificDates: string[] };
```

- **`customDays`**: Only exists if `type:"custom"`. An array that contains any combination of: `["monday", "tuesday", "wednesday", "thursday", "friday", "saturday", "sunday"]`.
- **`specificDates`**: Only exists if `type:"specific_dates"`. An array of discrete date strings universally formatted as `YYYY-MM-DD` (e.g., `["2026-05-15", "2026-12-25"]`). These are one-off reminders for explicit calendar dates.

### 2.3 The `times` Field (Array of TimeConfig)
Defines an array of exact times when the reminder should be triggered on its active days. It supports both 12-hour and 24-hour clocks for frontend rendering flexibility, but the executing agent must synthesize these fields to evaluate the absolute dispatch time.
Users can specify one or multiple times per reminder.

```typescript
type TimeConfig =
  | { format: "24h", hour: number, minute: number }
  | { format: "12h", hour: number, minute: number, period: "AM" | "PM" };
```

- **`hour` constraints**:
  - If `format:"24h"`, `hour` is between `0` and `23`.
  - If `format:"12h"`, `hour` is between `1` and `12`.
- **`minute` constraints**: Valid ranges are `0` through `59`.
- **Logic Mapping**: 
  - `12h/12:00/AM` -> `00:00`
  - `12h/1:00/PM` -> `13:00`
  - `12h/12:00/PM` -> `12:00` (Noon)

### 2.4 The `contacts` Field
- **Type**: `Array<string>` OR `undefined`
- **Behavior**: An array of string phone numbers (digits only, e.g., `"1234567890"` - no `+` prefix). 
- **CRITICAL NOTE**: If a user creates a reminder with *0 contacts*, **Firebase natively omits the entire key from the payload**. An executing agent parsing the payloads must assume `payload.contacts || []` to prevent `undefined` TypeError crashes.

### 2.5 The `createdAt` Field
- **Type**: `string` 
- **Behavior**: An ISO 8601 UTC timestamp identifying when the user saved this reminder. Example: `"2026-03-01T06:42:05Z"`.

## 3. Reference Example

This is a comprehensive JSON mock demonstrating a complex instance retrieved perfectly from the database:

```json
{
  "-Omc85vAVu9mrwKCTIAE": {
    "message": "Project Milestone Due",
    "date": {
      "type": "specific_dates",
      "specificDates": [
        "2026-04-01",
        "2026-05-01",
        "2026-06-01"
      ]
    },
    "times": [
      {
        "format": "24h",
        "hour": 23,
        "minute": 59
      },
      {
        "format": "12h",
        "hour": 8,
        "minute": 0,
        "period": "AM"
      }
    ],
    // Note: contacts is omitted from this mock because it is 0-length
    "createdAt": "2026-03-01T12:04:15Z"
  }
}
```
